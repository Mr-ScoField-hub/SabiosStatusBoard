# Aya Sprint 5 Goals

## 1. Offline Emergency Mode + SOS Wakeword
- Owner: **Mr. ScoField**
- Goal: Offline alerts + SOS wakeword detection
- Steps:
- SMS-based alerts without internet
- Integrate voice wakeword recognition
- Test triggers offline
- Deliverables: Fully functional emergency system
- Success: Alerts reliably work offline

## 2. Self-Defense Lessons (Computer Vision / AR/VR)
- Owner: **Mr. Qwabe**
- Goal: Teach escape/defense moves
- Steps:
- AR/VR tutorials
- Pose detection via CV
- Gamified progress tracking
- Deliverables: Interactive self-defense module
- Success: Lessons complete with progress tracking

## 3. Disguise Mode
- Owner: **Ms. Motsamai**
- Goal: Design/implement disguise themes
- Steps:
- Themes: calculator, gallery, news app
- Hide safety tools
- Test usability & visual polish
- Deliverables: Functional disguise modes
- Success: Tools hidden but accessible

## 4. UX/UI Enhancements
- Owner: **Aya Experience Team (Ms. Mashele + Ms. Mphelo)**
- Goal: Improve onboarding, Listen/Talk Status, Profile
- Steps:
- NewUserForm UX improvements
- Aya Listen/Talk Status
- Profile page for editing info
- Deliverables: Polished UX and profile management
- Success: Smooth user experience

## 5. Full-Stack Social Status Feature (Hardcoded)
- Owner: **Mr. Ngubane**
- Goal: Temporary status feed
- Steps:
- Hardcode status system
- Add status (disappears on refresh)
- Deliverables: Functional status feed
- Success: Users can post/view status

## 6. Newsfeed CRUD
- Owner: **Mr. Mazibuko**
- Goal: Users can create/edit/delete posts
- Steps:
- Support videos, images, text posts
- Implement CRUD logic
- Deliverables: Functional newsfeed
- Success: Users can manage posts

## 7. Build & Host Backend + Co-Pilot Integration
- Owner: **Mr. Stakio**
- Goal: Host Node.js/Express backend and assist team
- Steps:
- Backend setup & endpoints
- Cloud hosting (AWS/Heroku)
- Act as co-pilot for all members
- Deliverables: Reliable backend
- Success: All app features supported
